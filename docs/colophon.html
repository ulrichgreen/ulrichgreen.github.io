Colophon

This prototype is built as a reading object first: static files in the docs folder, no framework, no analytics, no external assets, and only enough CSS and JavaScript to clarify the typography.

The choices on this page follow the direction already written down in this repository: authored HTML over abstraction, performance as part of the design, typography over graphics, and a visible respect for print, restraint, and maintenance.

Materials

The site is plain HTML, raw CSS, and a few lines of Vanilla JS. That matches the stated stack in TECH-STACK.md and the broader standards in SITE-PLANNING.md: low-level tools, static output, and nothing added just because the modern web expects a logo for it.

The current index page is a standalone prototype. It carries its own styles, its own controls, and its own content in one file. This colophon follows the same approach so it can be opened, read, printed, or archived without any extra machinery.

Typography

Typography is doing the heavy lifting here. The page uses a system serif stack, a 65ch measure, and an 8px baseline grid with line-heights of 24, 32, 40, and 56 pixels. Those are the same proportions the index page uses for body copy, headings, notes, and interface text.

This is consistent with the repo's larger argument: the craft should be visible in spacing, hierarchy, rhythm, and restraint before the site reaches for decorative graphics or heavier tooling.

Operating Principles

  • Static files are the product, not an intermediate artifact hidden behind a platform.
  • No third-party widgets, trackers, or hosted dependencies are required to read the page.
  • Dark mode, reduced motion, and print are treated as normal reading conditions.
  • The source should stay legible enough that the implementation can be inspected directly.

Related Notes in This Repo

The supporting documents expand the same ideas from different angles: TECH-STACK.md explains the low-level tool choices, CODE-EXAMPLES.md describes how code should be treated as typography, SITE-PLANNING.md sets the editorial and design standard, and SURPRISE-IDEAS.md explores what delight can look like without abandoning restraint.

Summary

The site should feel authored, not assembled. This colophon exists to make the hidden choices legible: system fonts, static HTML, raw CSS, minimal JS, zero external dependencies, and a preference for exactness over volume.
